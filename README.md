# Bash M. Money App

TypeScript pnpm workspace with:

- `artifacts/sacco-app` - Vite/React frontend
- `artifacts/api-server` - Express API
- `lib/db` - Drizzle/PostgreSQL schema
- `lib/api-client-react` and `lib/api-zod` - generated API client/schema packages

## Local Setup On Windows

From the workspace root in PowerShell:

```powershell
$env:COREPACK_HOME = "$PWD\.corepack"
New-Item -ItemType Directory -Force .corepack-shims | Out-Null
corepack enable --install-directory .corepack-shims
$env:PATH = "$PWD\.corepack-shims;$env:PATH"
corepack pnpm install
```

Copy `.env.example` to `.env` and set `DATABASE_URL` to a PostgreSQL database.

```env
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/bash_m_money
SESSION_SECRET=replace-with-a-long-random-secret
```

Push the schema to the database:

```powershell
corepack pnpm --filter @workspace/db run push
```

## Run The App

Start the API:

```powershell
corepack pnpm run dev:api
```

Start the web app in another PowerShell window:

```powershell
$env:COREPACK_HOME = "$PWD\.corepack"
$env:PATH = "$PWD\.corepack-shims;$env:PATH"
corepack pnpm run dev:web
```

## Deploy The Backend On Render

This repo includes `render.yaml`, which defines:

- Render Web Service: `bash-m-money-api`
- Render Postgres database: `bash-m-money-db`

Deploy steps:

1. Push this repository to GitHub.
2. In Render, create a new Blueprint and select this repository.
3. Render will provision Postgres and set `DATABASE_URL` automatically.
4. After deploy, open the web service URL and check `/api/healthz`.

Render should set these environment variables:

```env
NODE_ENV=production
LOG_LEVEL=info
SESSION_SECRET=<generated by Render>
DATABASE_URL=<from Render Postgres>
```

After Render gives you the API URL, rebuild the APK with:

```powershell
$env:VITE_API_BASE_URL = "https://bash-m-money-api.onrender.com"
corepack pnpm --filter @workspace/sacco-app run android:release
```

Replace the URL with the exact Render service URL if you choose a different service name.

## Mobile Builds

Android uses Capacitor. After the web app builds, Capacitor copies `dist/public` into the native Android project.

```powershell
corepack pnpm --filter @workspace/sacco-app run android:debug
```

The debug APK is created under `artifacts/sacco-app/android/app/build/outputs/apk/debug/`.

For iOS, Capacitor can generate/sync the iOS project, but the actual app package must be built on macOS with Xcode and an Apple Developer account. iOS apps are packaged as `.ipa`, not `.apk`.
